# Zero Action AI Coding Rules & Guidelines

Welcome, Agent. You are pair-programming on **Zero Action**, a production-quality AI Development Partner CLI. To ensure the codebase remains clean, fast, and bug-free, you **must** strictly adhere to the following rules:

---

## 1. Core Architecture Principles

- **CLI Layer (`zero/cli/`)**: Command routing, CLI argument parsing (using Typer). **Do not** write business logic here.
- **Service Layer (`zero/services/`)**: Core application components (Config, Logging, AI Service wrappers).
- **Memory Layer (`zero/memory/` and `zero/storage/`)**: Short-term session, decisions, and local semantic/RAG databases (backed by SQLite).
- **Repository Intelligence (`zero/repository/`)**: Workspace analysis, git metrics, and language detection.

---

## 2. Performance Rules (CRITICAL)

- **Snappy CLI Startup (< 300ms)**: Zero Action must boot and display the help menu in under 300ms.
- **Lazy Imports**: Heavy modules (like `AIService`, `MemoryManager`, `git`, `sqlite3`, `litellm`) **must not** be imported globally in CLI command files. Import them locally inside the body of Typer subcommand functions.
- **PEP 484 Static Type Checking**: Use `from typing import TYPE_CHECKING` to keep imports deferred for type checkers without loading them at runtime.

---

## 3. Code Quality & Formatting Guidelines

- **Ruff Linter**: The codebase must maintain 100% compliance with `ruff check`.
  - Always clean up unused imports (especially after refactoring).
  - Do not use unnecessary f-strings without placeholders (`F541`).
- **Mypy Type Checker**: The codebase must be 100% type-safe. Run `uv run mypy zero tests --ignore-missing-imports`.
  - Ensure all function signatures have complete type annotations.
  - Safe byte-to-str: Be careful when handling git summaries/messages. If a value can be `bytes`, decode it cleanly using `.decode("utf-8", errors="replace")`.
- **Pytest Suite**: All tests must pass cleanly. Run `uv run pytest`.

---

## 4. Key Agentic Commands Reference

- **`zero test` (Self-Healing)**: Runs tests/checks in the background. If they fail, parses log traces, finds failing files, queries AI for a diff fix, applies it, and re-runs tests up to `--max-iterations`.
- **`zero pr` (Git Auto-Pilot)**: Analyzes git diffs, checks out a new branch, commits using **Conventional Commits** (generated by AI), pushes to `origin`, and creates a PR (via GitHub CLI `gh` or fallback compare URL).
- **`zero init`**: Analyzes directory context and populates local RAG indexes.
- **`zero chat`**: REPL chat environment supporting Claude Code style slash commands (`/fix`, `/test`, `/pr`, etc.).
