Thanks for contributing! Please follow these guidelines:

1. Fork the repo, create a feature branch, and open a PR against main.
2. Write tests for new features and run the full test suite.
3. Keep changes small and focused; follow Conventional Commits.
4. Run linters (ruff) and type checks (mypy) locally before submitting.

For design decisions or major refactors, open an issue first to discuss scope and approach.
